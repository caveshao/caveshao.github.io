【翻译】GitHub 的光明未来

By caveshao

发表于 2022-06-06

文章目录

作者: defunkt ,GitHub CEO & 联合创始人

=> 点击看原文 <=

我很高兴宣布，微软正在收购 GitHub，预计协议将在年底前完成。我们想尽快分享这条消息，尽管离完全结束还需要几个月。

当十年前 GitHub 首次推出时，我从未想过这个头条新闻。Git 是一个功能强大但是利基的工具，就像云只是天空中的东西，而微软是一个非常不同的公司。当时的人说，开源与商业混合起来就像石油与水。

我们不同意这个观点。作为开发者，我们知道这是一个错误的二分法——长期以来我们在商业环境中成功地使用了开源软件。我们需要的是更简单的与他人合作的方式，不管代码是私人的、公开的或者介于两者之间。 我们想用 Git 来做这件事，我们想要世界上的其他人能够加入进来，并且我们不希望它花费一毛钱，倘若它是开源的。因此我们创建了 GitHub。

当然，现在许多事不同了。Git 是最受欢迎的版本管理系统，云主要是电脑，微软是全球最活跃的 GitHub 组织。微软的 vscode 项目被数以百万计的开发者喜爱，这个项目记住 GitHub 的平台构建且完全开源。除此之外，现在主要的企业普遍拥抱开源。世界已经意识到高效、快乐开发人员的重要性。同样，人们现在都拥有智能手机。

然而，没变的是我们对开发者的关注。从一开始，我们着迷于为使用它的人们创造一个产品。我们想让开发者更加高效，我们想让更多的人成为开发者。从“云编程与边缘编程”，GitHub 的任务是不管经验水平帮助每一个开发者有效的学习、编程构造软件。

因此，当我们展望未来十年及其后的软件开发，我们知道这些都与开发者有关。我们了解了他们的主意，正如在过去几年时间通过 Git LFS 到 Electron 项目的合作了解了微软这个团队一样。他们在开源领域的工作激励了我们，我的世界与领英的成功收购向我们展示了他们非常重视新业务，Azure 的发展证明了他们是一个创新的开发平台。

但不止如此，他们对未来的远景与我们的愿景紧密相关。我们都认为 GitHub 需要为所有开发人员保持一个开放的平台。无论你的语言，技术栈，平台，云，或许可如何，GitHub 将永远是你创造软件，合作，造轮子最好的地方。

我们都认为软件开发需要更轻松，更易用，更智能，更开放，所以更多的人可以成为开发者而已存的开发者可以话更多的时间关注他们尝试解决的独一无二的问题。

我们都看到开发者的需求日益增长以及软件在我们生活的方方面面扮演者越来越重要的角色。

并且，更重要的是，我们都相信我们可以在一起做更重要的事。毕竟，合作是我们做的每件事的核心。

作为这次改变的一部分，纳特·弗里曼将会担任 GitHub 的 CEO。我们花了一些时间寻找新的 CEO，找到了微软和纳特，我们相信在未来几年 GitHub 的社区和公司将会更加强大。纳特合伙创办了 Xamarin 并且多年间一直致力于许多开源项目，有许多软件与开源软件社区的经验，是继续提高开发者环境、帮助 GitHub 成长最合适的人。

对于我来说，我将在微软担任与纳特和团队工作接近的新角色，未来我会分享更多细节。

我为我们团队和 GitHub 在过去十年间完成的事感到感到自豪，我迫不及待地想看看未来会发生什么。软件开发的未来是光明的，我很高兴能与微软联手实现这一目标。

@defunkt
CEO & 联合创始人 ,GitHub
